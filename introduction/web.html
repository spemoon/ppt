什么是XHTML

  • XHTML 是 The Extensible HyperText Markup Language 可扩展超文本置标语言的缩写

  • XML 虽然数据格式化能力强大，完全可以替代 HTML，但面对以千万计算的已有的网站，全面采用 XML 阻力过大

  • 因此，我们在 HTML 4.01 的基础上，用 XML 的规则对 HTML 重新定义，就得到了 XHTML

XHTML 的思想

  • 简单地用 XML 的语法对 HTML4 进行改写：

    • 利用 DTD 对语法进行规定
    • 标记名称统一为小写
    • 标记属性改用小写
    • 属性值用引号括起来
    • 标记必须配对
    • img, br 和 meta 这样的标记带斜杠自关闭
    • ……

  • 具有严格的 XML 语法，更规范，易于学习和上手

XHTML 的特点

  • XHTML 具有 XML 的一切特点
  • 同时又保留了 HTML 的优越性
  • 提供了良好的可伸缩性，可大可小
  • 可大：能够自定义标记
  • 可小：HTML 有90多个标记，XHTML 可以根据客户端设备的不同决定支持哪些标记

注意

所谓 XHTML 就是用 XML 重新定义的 HTML

用原来 HTML 的标记名作为 XML 里的标记

XHTML 还是 XML!

一个 XHTML 文档也是有效的 XML 文档

只不过长得很像 HTML~

XHTML 的作用在于定义文档的整体结构，而不是定义显示细节

它是结构层技术

因此你写的 XHTML 文件可能看起来很丑

建立 XHTML 的目的就是实现 HTML 向 XML 的平稳过渡

XHTML 的版本

XHTML 1.0 (2000)

http://www.w3.org/TR/xhtml1

XHTML 1.1 (2001)

兼容 1.0，小幅改变

目前推荐遵循的是 W3C 正式通过的 XHTML 1.0 以及 XHTML 1.1 标准

未来的标记语言

XHTML 2 (Draft)

语法不兼容 XHTML 1.x 和 HTML，改动较大

W3C 比较激进的改革尝试，事实证明接受度太低。W3C 决定集中力量发展 HTML 5，今年年底工作组解散

HTML 5 (Draft)

2004 年由 WHATWG 提出，于 2007 年被 W3C接纳，并成立了新的 HTML 工作团队

HTML 5 支持传统的 HTML 语法，也支持 XML 的语法

使用 XML 语法书写时也称为 XHTML5

XML 声明

虽然看起来加上XML声明是个比较好的做法，但现阶段由于一些浏览器软件的不完善，建议不要加上XML声明

主要是 IE6，加上XML 声明，浏览器会采用Quirks Mode 工作 (非正常工作模式)，造成页面显示问题

文档类型定义

DOCTYPE 声明必须放在每一个 XHTML 文档最顶部，在所有数据和标记之上（在 XML 声明之下）

XHTML 文件的 DOCTYPE 声明是必须的！

回顾引用公共 DTD 的形式：

<!DOCTYPE 根元素 PUBLIC "DTD 名称" "外部 DTD 的URL">

XHTML 1.0 的三种文档类型

XHTML 1.0 提供了三种 DTD 声明可供选择：

Transitional

要求比较宽松的 DTD，它允许你继续使用 HTML 4.01 中的标记 (但是要符合 XHTML 的写法)

  • 方便原来使用 HTML 的设计师过渡到 XHTML
  • 可以方便地将原先的 HTML 文档转换成符合 XHTML 的文档
  • 对于我们新时代的开发者，基本可以忘掉
  • <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

Frameset

专门针对框架页面设计使用的 DTD (不推荐使用)

  • 文档中要用到框架(Frameset)结构时使用
  • 现代网页设计不推荐使用框架！
  • 可以直接忘掉～
  • &li;!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Frameset//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-frameset.dtd">

Strict

要求严格的 DTD，不能使用任何表示信息样式的标识和属性，如居中 <center>

  • 与 XHTML 1.1 非常接近，推荐使用
  • 要求严格，不能使用任何 <center> 这样的标识和属性
  • <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
    • 在Strict声明下不能使用的标签
    • center
    • font
    • iframe
    • strike
    • u
    • applet
    • ……

加上 DOCTYPE 的好处

让网页以你期待的方式显示

以Standards Mode显示，而不是 Quirks Mode

让你的网页显示得更快！

Standards Mode渲染速度要比 Quirks Mode 快

XHTML常用的标记

头元素中的标记

  • <title>文档标题
  • <base>文档的基础URL
  • <link> 定义文档与其他文档的关系
  • <meta>定义文档的附加描述信息
  • <script>定义文档用的脚本链接，如Javascript
  • <style>定义文档用到的样式表链接，如CSS

基本标记 Basic Tags

<h1> to <h6>：标题一到标题六

<p>：段落

<br />：换行

<hr />：水平线

<br /> 这个方法已经严重被滥用来划分段落，它不应该使用在预计分段的两段文本中。是什么结构，该用什么标记就用什么标记，不是为断行而分段或者为分段而断行因为没有任何东西在换行标记之间，所以没有闭合标记，它是用“/”在br后自闭合的
